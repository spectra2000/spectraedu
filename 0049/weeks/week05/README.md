<h2>05. State ve Akış Kontrolü</h2>

<h3>Dersin Amacı</h3>
<p>
Bu haftada öğrenciler Streamlit uygulamalarında durum (state)
kavramını öğrenecek, kullanıcı etkileşimleri sırasında verinin
nasıl saklandığını ve uygulama akışının nasıl kontrol edildiğini
anlayacaklardır.
</p>

    <li>State Kullanım Örnekleri</li>
    <li>Ders Slaytları</li>
</ul>

<h3>Dış Kaynaklar</h3>
<ul>
    <li>Streamlit Session State API Referansı</li>
    <li>Streamlit Rerun Mantığı</li>
    <li>State Management Temel Kavramları</li>
</ul>

<h3>Uygulamalar</h3>
<ul>
    <li><code>st.session_state</code> ile veri saklama</li>
    <li>Buton tıklamalarını takip etme</li>
    <li>Kullanıcı etkileşimine göre ekran içeriğini değiştirme</li>
    <li>Basit adım (step) kontrolü uygulaması geliştirme</li>
    <li>
        Kullanıcının yaptığı seçimleri hatırlayan
        çok adımlı bir Streamlit paket programı oluşturulması
    </li>
    <li>
        Kullanılan temel bileşenler:
        <code>st.session_state</code>,
        <code>st.button</code>,
        <code>st.text_input</code>,
        <code>st.write</code>
    </li>
</ul>

<h3>Ödev</h3>
<ul>
    <li>
        <strong>Ödev 5:</strong>
        Streamlit kullanarak en az iki adımlı,
        kullanıcı durumunu hatırlayan bir paket program geliştiriniz.
        Uygulama, kullanıcı etkileşimlerine göre farklı ekranlar
        göstermelidir.
    </li>
</ul>
