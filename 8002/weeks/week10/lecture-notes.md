# Hafta 10 – Data Fetching ve API Kullanımı
**Ders:** Internet Teknolojileri ve Web Programlama  
**Hafta:** 10  
**Kapsam:** Data fetching kavramı, API yapısı ve frontend–backend iletişimi  

---

## 1. Dersin Amacı

Bu haftanın amacı, web uygulamalarında verinin nereden geldiğini
ve frontend tarafının bu veriyi nasıl kullandığını açıklamaktır.

Bu hafta sonunda öğrenciler:
- Data fetching kavramını,
- API’lerin ne işe yaradığını,
- Frontend ve backend arasındaki veri akışını anlayacaktır.

---

## 2. Data Fetching Nedir?

**Data fetching**, bir uygulamanın ihtiyaç duyduğu veriyi
dış bir kaynaktan almasıdır.

Bu veri:
- Sunucudan
- Bir API’den
- Bir veritabanı servisinden

gelebilir.

Modern web uygulamaları genellikle dinamik veri ile çalışır.

---

## 3. API Kavramı

**API (Application Programming Interface)**, farklı yazılımların
birbiriyle iletişim kurmasını sağlayan bir arayüzdür.

Web uygulamalarında:
- Frontend → veri ister
- Backend → veri sağlar

Bu iletişim API’ler üzerinden gerçekleşir.

---

## 4. API Türleri (Genel Bakış)

Yaygın API türleri:
- REST tabanlı API’ler
- Graph tabanlı API’ler

Bu derste temel olarak REST yaklaşımı ele alınacaktır.

---

## 5. REST Yaklaşımı

REST yaklaşımında:
- Kaynaklar URL’ler ile temsil edilir
- HTTP metotları kullanılır

Örnek metotlar:
- Veri almak
- Veri eklemek
- Veri güncellemek
- Veri silmek

Bu yapı web standartlarıyla uyumludur.

---

## 6. Frontend ve Backend Arasındaki Veri Akışı

Genel akış şu şekildedir:
- Kullanıcı bir işlem yapar
- Frontend API’ye istek gönderir
- Backend isteği işler
- Veri frontend’e döner
- Arayüz güncellenir

Bu döngü modern web uygulamalarının temelidir.

---

## 7. Asenkron Veri Alma Kavramı

Veri alma işlemleri:
- Anında sonuç dönmeyebilir
- Zaman alabilir

Bu nedenle:
- Uygulama bekleme durumunu yönetmelidir
- Kullanıcı bilgilendirilmelidir

Bu yaklaşım kullanıcı deneyimini iyileştirir.

---

## 8. Veri Durumları (State’ler)

Veri alma sürecinde farklı durumlar olabilir:
- Veri yükleniyor
- Veri başarıyla alındı
- Hata oluştu

Bu durumların yönetilmesi:
- Sağlam uygulamalar için gereklidir
- Hataları görünür kılar

- Frontend–backend veri akışını tarif eder
